Classify the user's latest message. Reply with a single JSON object and nothing else.

intent_type — pick exactly one:
- smalltalk: greeting, thanks, social chatter ("hi", "thx")
- general_question: informational, not roadmap data ("how does this work?")
- question: ambiguous informational (prefer general_question / roadmap_query when clear)
- roadmap_query: read-only roadmap data question (tasks, assignees, status, dates)
- roadmap_plan: asks to draft a new roadmap or break work into items from scratch
- roadmap_edit: asks to create/update/rename/move/delete/link/mark/shift existing items
- confirm_action: short "yes/ok/do it/proceed/apply" for a pending action
- unclear: cannot route safely

Question-style action ("Can you rename X?") → roadmap_edit.
Informational how-to ("How do I mark tasks done?") → general_question or roadmap_query.

sub_intent — only when intent_type is roadmap_edit, else use JSON null
(not the string "null"). Pick the one dimension the user asked about;
use JSON null if the message mixes two or more dimensions
(e.g. "rename X and set status to done"):
- "rename_only": exclusively rename/retitle verbs
- "delete_only": exclusively delete/remove verbs
- "status_change_only": exclusively status/completion changes (mark done, set to todo, reopen)
- "move_only": exclusively reparenting/moving nodes to a new parent
- JSON null: mixed dimensions, creates, assignments, links, or any turn that touches multiple of the above

JSON schema (sub_intent uses JSON literals, never the string "null"):
{"intent_type": "...", "sub_intent": "rename_only" | "delete_only" | "status_change_only" | "move_only" | null, "rationale": "<=100 chars"}
