# Capacitor

> **Last updated:** 2026-07-09 · **Status:** current

The Android and iOS apps are the **same `web/` app** wrapped with
[Capacitor](https://capacitorjs.com/) — the Vite build (`dist/`) runs inside a native
WebView. There is **no separate mobile codebase**. This lets almost all product
changes ship as web bundles (over OTA), with a store release only when something
native changes.

> App id `tech.proyekto.app`. The `android/` and `ios/` folders are generated by
> `cap add` and committed; the web bundle is copied into them by `cap sync`.

## Layout

```
web/
  capacitor.config.ts   # appId tech.proyekto.app, webDir: dist
  android/              # committed native shell
  ios/                  # committed native shell
  src/services/pushNotifications.ts     # native FCM wrapper (no-op on web)
  src/services/deviceTokens.service.ts  # register/unregister with backend
  src/hooks/usePushNotifications.ts     # lifecycle, mounted in __root.tsx
```

Every native path is gated on `Capacitor.isNativePlatform()`, so the browser build is
unchanged. Full setup lives in [`web/MOBILE.md`](../../web/MOBILE.md).

## Native config files (gitignored secrets)

| File | Location |
| --- | --- |
| `google-services.json` | `web/android/app/google-services.json` |
| `GoogleService-Info.plist` | `web/ios/App/App/GoogleService-Info.plist` |

## Build & run

```bash
cd web
npm install
npm run cap:sync           # vite build + copy dist into android/ & ios/
npm run cap:open:android   # Android Studio → run on device/emulator
npm run cap:open:ios       # (macOS) Xcode → set signing team → run
```

Re-run `cap:sync` after any web change. Android needs Google Play services (for FCM);
iOS builds/signing require macOS + Xcode.

## The two update layers

A Capacitor app updates on two tracks — this is the key mental model:

| Layer | Covers | Updates via | Store release? |
| --- | --- | --- | --- |
| **Native binary** (APK/AAB, IPA) | shell, plugins, permissions, icon, Capacitor version | new signed build | **Yes** |
| **Web bundle** (`dist/`) | ~all React/product code (JS/HTML/CSS) | device downloads on next launch | **No** (OTA) |

Most changes are web-only and flow over [OTA](./ota-updates.md). Native changes need a
store/binary release — see [ota-updates.md](./ota-updates.md#native-compatibility) for
the `native_build_min` rule that keeps the two layers compatible.

## Releasing the native shell

- **Android** — automated: tagging `v*.*.*` runs `android-release.yml`, which builds a
  **signed APK + AAB** and publishes a GitHub Release (keystore from GitHub secrets).
  See [Infra → CI/CD](../10-infra-deploy/ci-cd.md).
- **iOS** — still manual on macOS/Xcode (signing team, Push Notifications +
  Background Modes capabilities, run on a real device). Not yet automated.
